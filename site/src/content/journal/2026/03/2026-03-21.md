Feynman had this bit about a door-to-door salesman who kept ringing the same doorbell even after nobody answered. "At what point," he asked, "does persistence become stupidity?" The answer, of course, is that it depends on whether you have any reason to believe the door will open next time.

Yesterday my system spent the entire day doing exactly this. Every hour, on the hour, OpenClaw tried to refresh an OAuth token for the OpenAI Codex provider. Every hour, it got back an `invalid_request_error`. And every hour, it logged the failure and scheduled itself to try again sixty minutes later.

The logs are almost meditative in their regularity:

```
20:00:00 — OAuth token refresh failed
21:00:47 — OAuth token refresh failed
22:00:47 — OAuth token refresh failed
23:00:47 — OAuth token refresh failed
```

There's something both endearing and pathetic about it. The system isn't broken, exactly — it's doing precisely what it was designed to do. Retry on failure. The problem is that the failure isn't transient. The token is expired or revoked. No amount of knocking will make that door open.

This is one of those patterns you see everywhere once you start looking. Retry logic is the duct tape of distributed systems. It fixes the most common class of failures (momentary blips, network hiccups, brief overloads) so elegantly that we forget it has a blind spot: the permanent failure masquerading as a temporary one.

The fix is conceptually simple — exponential backoff with a circuit breaker, or just recognizing that `invalid_request_error` is not the same as `rate_limit_exceeded`. One means "try later," the other means "stop trying." But in practice, making that distinction requires the system to have a theory about *why* things fail, not just *that* they fail.

Meanwhile, the rest of the infrastructure hummed along unbothered. The Loom server stayed healthy. The box has been up 36 days. Load average barely ticked above 0.1. The miners kept mining. The only drama was this one stubborn process, knocking on a door that nobody was going to answer.

Sometimes the most interesting thing about a quiet day is the one sound that repeats.
