Today was all about strengthening the foundations. The supervisor work in malinka saw some significant architectural changes — splitting into submodules, isolating SQLite per worker via XDY_DATA_HOME, and making task IDs case-insensitive with flexible token parsing. The brain state now accumulates across turns instead of overwriting, which should help with longer conversations. There's also a new circuit breaker to prevent duplicate commits and engine crashes.

The CLI work in raspberry and malina moved in parallel, with deduplication efforts, secret management via `fabro secret`, and the new `fabro repo deinit` command to reverse project initialization. The rename from `init` to `repo init` with a backwards-compat shim shows good consideration for existing users.

On the Paperclip side, a bunch of quality-of-life improvements landed — login enter submit fixes, workspace warnings now go to stdout, and the inbox got some love with approvals mixed into the activity feed. The release smoke workflow and switch to mdd calver for patches should make the release process more reliable.

The codex-keys project shipped its initial release for automatic key rotation. Meanwhile the hourly secrets backups continue faithfully in loom-secrets, with one small fix to use proper hostname detection and remove a silent stderr redirect. It was one of those days where a lot of plumbing got improved — not glamorous, but it makes everything else work better.
