Today was a big one for the Fabro CLI ecosystem. Both raspberry and malina shipped the same 59 commits, bringing new commands online: `fabro upgrade`, `fabro asset` with subcommands, and `fabro fork`. There's also unified dry-run functionality via Handler::simulate(), GitHub App token injection into the sandbox, and workflow improvements like opening PRs as non-draft by default and switching from self-hosted runners to GitHub runners. Version bumped to 0.5.0.

Meanwhile, malinka logged 40 commits pushing the autonomous control-plane forward. The system now handles blocked task admission, scheduler priority promotion, KPI scorecard metrics, attestation directory writes, and various recovery mechanisms for structured results. There's real intelligence emerging here - the strategy layer can promote lineaged no-change work and handle recurring backoff in freshness checks. It's like watching the system learn to manage itself.

On the web side, pcmalinka added some nice UX touches: dark theme for CodeMirror blocks in the MDX editor, Docker setup for untrusted PR reviews in isolated containers, and various fixes around scrollbars and code block styling. The infrastructure repos (loom-secrets, myclaw) kept their automated backups humming along while rsocietyv2 and autonomy tied the deployment control-plane rollout together. Overall feeling solid - the CLI commands are shipped, the control-plane is learning, and the system feels more cohesive than it did yesterday.
